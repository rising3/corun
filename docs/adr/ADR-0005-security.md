## 概要

機密情報の安全な取り扱いと最小権限の原則を定める。

## コンテキスト

CLI ツールは外部サービスの API キーや資格情報を扱う可能性があり、漏洩防止の運用設計が必要である。

## 決定

- 機密情報はソースコードにハードコードしない。
- 環境変数またはシークレットマネージャ（例: GitHub Secrets）を利用する。
- 実行時は最小権限の原則を採用する。

## 決定の理由（Rationale）

- ハードコードは漏洩リスクを高めるため、セキュリティベストプラクティスに反する。

## 影響とフォローアップ

- ドキュメントとコードレビューで機密情報が含まれていないことを確認するプロセスを設ける。
- CI/CD とシークレットストアの利用方法をドキュメント化し、開発者向けガイドを作成する。
- 依存パッケージの脆弱性スキャンを定期実行すること。

## 代替案

- 簡易な設定ファイル（暗号化なし）での管理は可能だが、運用上の安全性が低下するため採用しない。
