# 概要

`corun` は Copilot CLI を補助する CLI ツール群の一部として設計されたユーティリティです。
目的は反復的なコマンド実行やタスク定義の自動化を容易にすることです。

主要コンポーネント:

- `bin/` : 実行可能スクリプト
- `src/` : 実装ソース
- `docs/` : ドキュメント（このディレクトリ）
- `docs/specs/features/` : 機能仕様（コマンド定義、フラグ、入出力など）

簡易の実行フロー:

1. ユーザーが `corun run` 等のコマンドを実行する
2. `corun` は指定されたタスク定義や引数を解釈し、必要な処理を順次実行する
3. ログや出力を `stdout`/ログディレクトリへ出力する

---

拡張性:

- 機能は `features` に仕様を追加して実装することで拡張できます。
- 将来的に `api/` や `db/` カテゴリを追加して、外部サービス連携やデータ永続化を扱えます。

---

## はじめに

このプロジェクトは、反復的なタスク実行とワークフロー自動化を簡潔に定義・実行できる CLI ツール群を提供することを目的としています。開発者や自動化エージェントが安全かつ再現可能にタスクを実行できる環境を目指します。

## 主な機能

- タスク定義の読み込みと実行（`run`）
- テンプレートや設定の生成（`gen`）
- ログ出力とレベル制御、実行の再現性を高めるオプション
- CI 連携用のコマンドセットとテストスイート

## 学習目標

- `corun` のコマンド構造とフラグ設計を理解する。
- タスク定義ファイルの作成と実行方法を習得する。
- ログやエラー出力を元に問題を特定し、再現可能な修正を行う。

## ターゲットオーディエンス

- CLI ツールや自動化ワークフローを組む開発者。
- Copilot / エージェントを統合して自動処理を行う研究者・エンジニア。
- CI/CD パイプラインに自動化コマンドを組み込みたい運用担当者。

## 成功指標

- 主要機能（`run`/`gen`）の基本動作がテストで安定して通ること。
- ドキュメントを参照して初見の開発者が 30 分以内に簡単なタスクを実行できること。
- CI 実行での再現性が確保され、エラー率が低いこと。

## プロジェクトロードマップ（高レベル）

1. 基本機能の安定化（`run`, `gen`, ロギング、テスト）
2. 仕様書・テンプレート整備とドキュメント充実
3. 外部連携（`api` カテゴリ）とデータ永続化（`db` カテゴリ）の導入
4. UX 改善（エラーメッセージ、ログの可読性向上）
